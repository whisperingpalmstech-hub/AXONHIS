2. Introduction 2.1 Purpose
This Functional Requirements Document (FRD) defines the functional and non-functional requirements for the Pharmacy Module (OP & IP Pharmacy) within the Hospital Information System (HIS). The Pharmacy Module is designed to manage medication dispensing, inventory control, billing, narcotics tracking, and comprehensive pharmaceutical services for both outpatient (OP) and inpatient (IP) settings. This document serves as a comprehensive guide for stakeholders, developers, and implementation teams.

2.4 Module Context within HIS
The Pharmacy Module is a critical component of the Hospital Information System (HIS), serving as the central hub for all pharmaceutical operations. It interfaces with multiple HIS modules:
OPD Module: Receives prescriptions from outpatient consultations
IPD Module: Manages medication orders for admitted patients
Doctor Desk: Receives medication prescriptions from physicians
Nursing Module: Coordinates medication administration and returns
Billing Module: Integrates for medication billing and payment processing
Inventory/Store Module: Synchronizes stock levels and procurement
Front Office: Coordinates patient flow and billing
EHR System: Maintains medication history and drug interaction alerts
Finance Module: Tracks revenue, discounts, and financial transactions
Reporting Module: Provides operational and compliance analytics
3. System Architecture and Integration
3.1 Module Architecture
The Pharmacy Module follows a three-tier architecture integrated within the HIMS ecosystem:
Presentation Layer: Web-based user interface accessible through HIMS portal
Business Logic Layer: Medication dispensing, inventory management, billing, and workflow engine
Data Layer: Integrated with HIMS central database for patient records, medication data, inventory, and billing

5. Functional Requirements
5.1 OP Pharmacy - Walk-In Sales
5.1.1: Walk-In Sales Management
The system shall support walk-in sales for customers without prior appointments:
Two modes: Patient Search (for registered patients) and Walk-In Sales toggle
Patient search by UHID, name, gender, mobile, DOB, identification type/number
Display visit number and doctor name for registered patients
Add medications as per prescription or Patient request
Manual instruction entry or dropdown selection
Stock availability check for each medication
Generic combination search
Add/remove medications dynamically
Add substitute medications
Clear field functionality
Payment entitlement selection (self, insurance, corporate)
Item kit support for bundled medications
At COP checkbox for cost-of-purchase pricing
Cash Settle checkbox for immediate payment settlement
Bill generation and settlement
Automatic stock deduction
5.1.2: Patient Search and Registration
The system shall enable patient search and walk-in registration:
Search registered patients by multiple criteria
Display patient demographics and visit history
Support for walk-in patient without registration
Quick registration for new walk-in customers
Integration with HIS patient master
5.1.3: Medication Selection and Dispensing
The system shall support comprehensive medication selection:
Medication search by name, generic name
Display medication details (strength, form, pack size)
Stock availability indicator
Batch and expiry date selection
Quantity entry with validation
Dosage and instruction capture
Drug interaction alerts
Allergy checking
Substitute medication suggestions
Non-formulary medication addition
5.2 OP Pharmacy - Sales Worklist
5.2.1: Sales Worklist Management
The system shall manage sales worklist from OPD nursing orders:
Receive medication orders from OPD Nursing Cover Sheet
Display patient queue with order details
View prescription details
Process medication dispensing
Add non-formulary medications
Status tracking (pending, in-progress, completed)
Bill settlement integration
Automatic entry in Bill List upon completion
5.3 OP Pharmacy - Sales Return
5.3.1: Sales Return Processing

Access settled bills for walk-in sales and sales worklist
Search and retrieve bills for return
Select medications for return
Enter return quantity
Specify return reason
Validate return eligibility (time limits, condition)
Process refund calculation
Generate return receipt
Update stock levels (add returned quantity)
Update billing records
Status change to completed upon processing
5.4 OP Pharmacy - Walk-In Prescription
5.4.1: Walk-In Prescription Management

6. Process Workflows
6.1 OP Pharmacy Workflow
Complete workflow for outpatient pharmacy:
1: Patient Registration/Walk-In
Patient visits pharmacy with prescription
Search registered patient or enable Walk-In Sales toggle
 Enter patient details if new
2: Prescription Entry
 Add medications from prescription
 Check stock availability
  Add substitutes if needed
  Add item kits if applicable
3: Billing
Apply discounts if applicable
Select payment entitlement
Calculate total amount
Generate bill
4: Payment
Select payment mode
Process payment
 Settle bill
 Generate receipt
 5: Dispensing
 Dispense medications
 Provide instructions to patient
 Update stock levels as per requirement 
6: Sales Return (if applicable)
 Patient returns medication
 Search bill in Sales Return
 Select medications to return
 Enter return quantity and reason
 Process refund
 Update stock and billing
6.2 IP Pharmacy Workflow
Complete workflow for inpatient pharmacy:
1: Medication Order
  Doctor/Nurse orders medication from IPD Nursing/Doctor Cover Sheet
  Order appears in IP Pharmacy Pending Issue
2: Order Processing
   Pharmacist reviews order in Pending Issue
  Check patient details and medication list
  Verify stock availability
  Add substitutes if needed
   Add non-formulary medications if required
 3: Dispensing
  Enter dispensed quantity
  Apply discounts if applicable
  Generate pharmacy receipt
  Status changes to completed
4: Billing
  Charges reflected to IPD bill
  View in IPD Charges Bill List
  Bill settlement at discharge
 5: Patient Issue Acceptance
  Nursing staff views issued medications
  Accept medications for administration
  Or reject with reason if clinically inappropriate 
  Status changes to closed

6: Patient Issue Rejection (if applicable)
Pharmacy reviews rejected medications
 Accept rejection or reject rejection
 Bill adjusted for accepted rejections
 Refund processed
  Stock updated
7: Patient Issue Return (if applicable)
 Nursing initiates return for unused/wrong medications
  Enter return quantity and reason
  Pharmacy accepts return
  Generate return receipt
  Bill adjusted
  Stock updatedIn Patient Pharmacy Flow Nursing-> Pharmacy-> Patient Indent -> Add New Items -> SAVE Pharmacy -> IP Pharmacy -> Patient Consumable Issue -> Nursing Issue Acceptance
6.3 Narcotics Management Workflow
Strict workflow for controlled substances:
1: Narcotics Order
Doctor orders narcotics from IPD Nursing Cover Sheet
 Medication must have "Is Narcotics" flag enabled
 Order appears in Narcotics Store Issuance Worklist
 2: Validation
  Narcotics In-charge reviews order
  Validate prescription and patient details
  Verify regulatory compliance
  Approve and save
  Status changes to closed
Step 3: Delivery
 Order appears in Delivery Worklist with completed status
  Dispense narcotics to nursing staff
 Document handover
4: Administration
  Nursing staff administers narcotics
  Update Drug Administration record
  Document time and dosage

 5: Empty Ampoule Return
  Nursing returns empty ampoules
  Entry in Return Empty Ampoules with pending status
 Pharmacy accepts empty ampoules
 Complete audit trail maintained
  Status changes to completed
6.4 Return and Rejection Workflow
Workflow for medication returns and rejections:
 1: Initiation
 Nursing staff identifies medication to return/reject
  Select medication and enter quantity
  Select reason from dropdown
  Save (status: pending)
2: Pharmacy Review
Pharmacist reviews return/rejection request
 Verify reason and quantity
 Check medication condition
3: Decision
  Accept return/rejection
  Or reject return/rejection with justification
  Enter accepted quantity
4: Processing
 Generate return receipt
  Update stock levels
  Adjust patient bill
  Process refund or credit
  Status changes to completed

5: Billing Update
 Returned amount deducted from IPD bill
  Refund reflected in final bill
  Financial records updated
